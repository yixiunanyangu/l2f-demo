The Challenge

Why Traditional Cropping Fails

Conventional cropping treats GUI as a flat pixel grid. When multiple similar elements exist across overlapping windows, models get hopelessly confused.

INSTRUCTION
Same task for both methods: “Close the File Explorer window.”
❌ Traditional Local Cropping

Coarse Localization → Ambiguous Crop

Model roughly locates the target area, expands a fixed bbox around it — but the box captures close buttons from multiple windows. Prediction fails.

✅ L2F — Logical Region (WLR)

Window Lock → Precise Grounding

Model first identifies the containing window (WLR), then searches only within it. Single unambiguous close button found instantly.

Annotation System

Three-Level Annotation Strategy

L2F covers every GUI type through three complementary levels. Distribution: WLR ≈ 50% · WSLR ≈ 40% · ELR ≈ 10%

WLR

Window-Level Logical Region

Multi-window environments · Task: "Save the file open in WPS"

For multi-window desktops: identify the specific application window containing the target, annotate that window's entire boundary as the logical region.

  • ①
    Locate the window containing the target
  • ②
    If window > 600×600: sub-divide within it
  • ③
    Occluded window → annotate full visible portion
  • ④
    Small window → expand, avoid contrast boundaries
WSLR

Workspace-Level Logical Region

Single-window multi-panel · Task: "Debug in VSCode"

For single-window apps: segment the interface into functional panels, annotate the workspace panel where the target task point resides.

  • ①
    Identify all functional panels within the window
  • ②
    Panel < 600×600: expand away from unrelated areas
  • ③
    Strong color-contrast boundary → don't cross it
  • ④
    Elongated panel → proportional expansion
ELR

Element-Level Logical Region

Simple / sparse interfaces · Task: "Turn off the clock at 4:30 pm"

For simple, sparse interfaces with no distinct workspace subdivisions. Centers the target element, minimizes distracting context.

  • ①
    No distinct workspace panels → use ELR
  • ②
    Center the target element in the region
  • ③
    Minimize surrounding context clutter
  • ④
    Expand slightly if region < 600×600

Regional-Constraint Reward RC
R_C = exp( −(S_pred − S_ideal)² / 2(Λ₀ · S_ideal)² )

Gaussian penalty. Λ₀ = 0.25. Reward → 1 when predicted area ≈ Sideal; decays exponentially as deviation grows.

Why Constraints Matter

Without area constraints, models exploit trivial strategies — predicting the whole image as "logical region" satisfies inclusion rewards without providing grounding benefit. The Gaussian penalty forces genuinely focused region predictions.

Annotation Pipeline

Annotator Decision Flow

1

Interface Assessment

Multi-window desktop? Single-window app? Sparse interface? Count windows and functional areas.

2

Level Selection

Multi-window → WLR. Single-window + distinct panels → WSLR. Sparse / no panels → ELR.

3

Bounding Box

Draw box containing the target. Verify: must include target point. Check: ≥600×600 px, ≤1024×1024 px.

4

Adjustment Rules

Too small? Expand carefully. Elongated? Balance proportions. Strong visual boundary? Don't cross it.

Edge Cases

Fine-Grained Annotation Rules

Nuanced decision logic distinguishing high-quality L2F annotations from arbitrary bounding boxes.

WLR Window-Level — 4 Special Cases
①
Occluded window: Target in a partially-hidden window → annotate the entire visible portion. Preserves window-level context even when partially hidden by another window.
②
Large independent window (>600×600): Apply WLR-style decomposition inside — identify functional sub-regions within it and annotate the most relevant sub-area rather than the entire window.
③
Decomposable but size-limited window: Further cropping would lose essential context → retain the full window as the logical region even if sub-regions can be identified.
④
Too-small window (expand required): May expand slightly for ≥600×600 compliance. Exception: if expansion crosses a visually distinct boundary (e.g., sharp color discontinuity like the white-to-dark-pink transition in Fig. 2 example 4), retain original window — avoid introducing irrelevant content.
WSLR Workspace-Level — 3 Adjustment Rules
①
Elongated workspace: Narrow strips bias the model toward directional reasoning. Extend the bounding box toward the direction with fewer distractors to achieve a more balanced aspect ratio.
②
Expansion constrained by image boundary: When the image edge limits expansion direction, prioritize a clean, well-defined region edge. Avoid expanding into visually complex or cluttered neighboring areas.
③
Small workspace with low visual contrast: When the sub-region blends visually with surroundings, moderately expand the bounding box to enhance contextual consistency and improve downstream detection robustness.
ELR Element-Level — Core Principles
①
Minimize surrounding interference: ELR prioritizes reducing interference from adjacent elements. The target point should be well-centered within the annotated region.
②
Focused, uncluttered context: Unlike WLR/WSLR which preserve broader functional context, ELR tightly frames the target element itself as the entire logical focus region.
③
When to use ELR: (a) Only one functional region exists; (b) Elements are sparse and clearly separated; (c) Interface has already been pre-cropped; (d) Task is equivalent to local grounding due to isolated target placement.
ALL Cross-Level Consistency Rules
📏
Size bounds (hard constraint): All logical region boxes: 600×600 ≤ box ≤ 1024×1024 pixels. Empirically validated range providing sufficient context while maintaining discriminability.
🎯
Containment guarantee (hard constraint): The logical region must always contain the ground-truth target point. Enforced by annotation guidelines AND the Logic-Consistency Reward R_ℓ during RL training.
🧠
Human perceptual alignment: Every annotation should reflect how a human would identify the relevant functional context before searching for the target — not just geometric proximity.
📊
Icon-text balance in region: Annotations follow a ~1:2 icon-to-text ratio in task targets, intentionally raising icon proportion to strengthen symbolic visual reasoning in the trained model.

Reward Function Design
R_total = R_ℓ · R_iou + w_C · R_C + w_p · R_p
R_ℓ: containment (target ∈ predicted box)
R_iou: region overlap with ground truth
R_C: area size constraint (Gaussian)
R_p: task-point accuracy (E2E only)
